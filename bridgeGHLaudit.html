What this means for Phase 2

Six architecture decisions, locked.

The audit gives us enough information to commit to specific design choices for each Phase 2 workflow. These are not guesses — they map to fields and tags that exist and are populated.

01

Entry filter for every Phase 2 workflow

One filter block at the top of every workflow. Med-spa scope confusion is gone because there's no med-spa to filter for, but service-line specificity still matters for branching.

# Required to enter: HAS tag in [physical therapy, pitcher/thrower, recovery/arm care, strength & conditioning (bb/sb), strength & conditioning (non-bb/sb)] AND active = "Yes" AND patient_number IS NOT empty # Automatic exit: HAS tag in [bridge therapy not interested, unqualified / not a good fit, spam likely]
02

Post-Eval Workflow trigger

Triggers off Jane data syncing into GHL, not GHL calendar events (since evals don't live in GHL calendars).

# Trigger: Tag "arrived" added (after eval session marked complete in Jane) OR last_appt_state changes to "arrived" # Branch by service: IF last_appt_treatment_name CONTAINS "Eval" → fire eval-specific flow ELSE → standard post-session flow
03

Retest Reminder logic

Cadence per service line still requires Victor's handwritten journey notes (the May 6 ask). Field-level wiring is locked.

# Trigger (daily check): last_visit < today - X weeks # X varies by service tag AND active = "Yes" AND NO tag "scheduled" # already booked, suppress # Service-specific cadence (pending Victor input): recovery/arm care → milestone-based (not time) pitcher/thrower → ~X weeks strength & conditioning → ~X weeks
04

Client Reactivation Sequence

Use `bridge therapy past client` tag as primary signal — more reliable than `discharged_at` field across service types.

# Trigger: last_visit > today - 30 days AND active = "No" OR HAS tag "bridge therapy past client" AND marketing_emails = "Yes" AND no_show_count < 3 # suppress chronic no-shows
05

Google Review Automation

Three possible milestone triggers, all on real fields. Victor picks which ones fire.

# Eligible milestones (any/all): appointments_count = 5 # 5th session OR HAS tag "arrived" AND last_appt_treatment_name CONTAINS "Eval" # post-eval OR discharged_at = today - 1 day # 24hr post-discharge # Compliance: AND ratings_emails = "Yes" AND no prior review-ask in last 90 days
